# goxeou 用户绑定手机号功能实施计划

> 版本: v1.0  
> 日期: 2026-05-31  
> 系统: goxeou 商城 (ThinkPHP 6 + MySQL)  
> 开发站: /mnt/data/12fz/wwwroot/goxeou_dev/

---

## 一、现状

| 模块 | 手机号字段 | 绑定功能 |
|------|-----------|---------|
| 会员 (`ddwx_member`) | 已有 `tel` 字段 | `ApiMy::settel()` / `settelsub()` 已实现，含短信验证码验证 |
| 后台用户 (`ddwx_admin_user`) | **无**手机号字段 | 无 |
| 短信发送 | `Sms.php` 已支持腾讯云短信 | 含验证码模板 |

## 二、需求

### 2.1 会员端（已有，可能缺前端UI）
- 会员个人中心 → 绑定/更换手机号
- 输入手机号 → 获取验证码 → 验证绑定
- 同一手机号不能重复绑定

### 2.2 后台管理端（需要新增）
- 后台用户表 `ddwx_admin_user` 增加 `tel` 字段
- 后台个人设置页面增加绑定手机号功能
- 管理员/商户可绑定/更换手机号

---

## 三、数据库变更

```sql
-- 后台用户表加手机号字段
ALTER TABLE `ddwx_admin_user`
  ADD COLUMN `tel` varchar(20) DEFAULT NULL COMMENT '绑定手机号' AFTER `un`,
  ADD INDEX `idx_tel` (`tel`);
```

---

## 四、需要修改的文件

### 4.1 后端

| 文件 | 修改内容 |
|------|---------|
| `app/controllers/User.php` | `save()` 方法增加 `tel` 字段保存；`edit()` 页面传递 `tel` 值 |
| `app/controllers/ApiMy.php` | `settel()`/`settelsub()`（会员端已有，无需改动） |
| `app/views/web_user/edit.html` | 后台用户编辑页增加手机号输入框 |
| `sms发送接口` | 检查现有 `Sms.php` 是否可用于验证码发送，如需要则添加发送验证码API |

### 4.2 前端UI（会员端）

需要确认会员端前端页面（H5/小程序）是否已有手机号绑定界面。如没有，需在会员个人中心增加。

---

## 五、实施步骤

### Step 1: 数据库加字段
执行 ALTER TABLE 给 `ddwx_admin_user` 加 `tel` 字段

### Step 2: 后台用户编辑页增加手机号
- `User.php` save 方法：增加 `$info['tel']` 写入
- `web_user/edit.html`：增加手机号输入框

### Step 3: 后台用户个人设置
- 增加后台用户个人中心设置手机号功能（绑定/更换）
- 可复用会员端的验证码发送逻辑

### Step 4: 验证测试
- 后台添加/编辑用户时填写手机号
- 会员端绑定手机号流程
- 手机号唯一性校验
