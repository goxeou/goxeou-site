# goxeou 商城直营店商品同步方案设计

> 版本: v1.2
> 日期: 2026-05-31
> 系统: goxeou 商城 (ThinkPHP 6 + MySQL)

---

## 一、背景

### 现状
- 每个商户完全独立，商品互不关联
- 商品表 `ddwx_shop_product` 通过 `bid` 字段关联商户
- 总店 `格希欧GOXEOU旗舰店`（id=20）
- 已标记直营店：惠水直营店(id=26)、长顺店(id=29)、都匀店(id=30)、镇宁店(id=31)

### 需求
1. 部分商户标记为**直营店**
2. **入住商户**（普通商户）保持原有功能不变
3. 直营店自动同步总店商品
4. 直营店可自主上架额外商品
5. **定价规则**（用户确认）：
   - 直营店可设置**整店调价百分比**（±%）
   - 直营店可**单独修改**某个同步商品的售价
   - 总店设置**最低保护价**，直营店售价不能低于此价

---

## 二、数据模型

### 2.1 ddwx_business 表字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | tinyint(1) DEFAULT 0 | 商户类型: **0=入住商户 1=直营店 2=总店** |
| `head_bid` | int DEFAULT NULL | 所属总店business.id |
| `price_adjust_percent` | decimal(5,2) DEFAULT 0.00 | 整店调价百分比（正=加价，负=降价） |

### 2.2 ddwx_shop_product 表字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `sync_from_bid` | int DEFAULT NULL | 同步来源总店bid |
| `source_pid` | int DEFAULT NULL | 源商品ID（总店原始商品ID） |
| `min_price` | decimal(11,2) DEFAULT 0.00 | 最低保护价（总店设置） |
| `override_price` | decimal(11,2) DEFAULT NULL | 直营店单独覆盖价（手动修改） |

---

## 三、定价逻辑

### 直营店商品最终售价计算

```
1. 优先取 override_price（直营店手动改价）
2. 否则按总店 sell_price × (1 + price_adjust_percent / 100)
3. 最终售价 = max(计算价, min_price)
```

### 优先级
**直营店手动改价 > 整店调价% > 最低保护价（保底）**

### 展示规则
- **直营店后台商品列表**：显示「总店原价、调价%、直营店售价、最低保护价」
- **前端消费者**：直接显示最终售价
- **总店后台商品编辑**：增加 `min_price` 字段

## 四、同步机制

### 4.1 总店新增商品 → 直营店复制
- `ShopProduct.save()` 末尾调用 `ShopSync::syncNewProduct($proid)`
- 遍历所有 `type=1 AND head_bid=总店bid` 的直营店
- 复制商品到直营店，记录 `sync_from_bid` 和 `source_pid`
- 直营店初始售价 = 总店售价 × (1 + 整店调价%)

### 4.2 总店修改商品 → 直营店同步
- `ShopProduct.save()` 末尾调用 `ShopSync::syncProductUpdate($proid)`
- 更新直营店同源商品的基础字段（名称、分类、图片、详情、市场价、规格数据）
- **同步**：`name`, `cid`, `cid2`, `pic`, `pics`, `detail`, `market_price`, `guigedata`, `paramdata`, `min_price`（最低保护价）
- **不覆盖**：`override_price`（直营店改价）、`stock`（库存）、`sales`（销量）

### 4.3 总店删除商品 → 直营店下架
- `ShopProduct.del()` 末尾调用 `ShopSync::syncProductDelete($proid)`
- 将直营店对应同步商品的 `status` 置为 0（下架），而非物理删除

### 4.4 手动同步
- 总店后台提供「同步所有直营店商品」按钮
- 批量调用同步逻辑，用于初始化或修复

## 五、功能模块

### 5.1 总店后台
1. **直营店管理** — 列表查看、设置调价%、解绑
2. **商品最低保护价** — 编辑商品时增加字段
3. **同步管理** — 一键同步按钮

### 5.2 直营店后台
1. **同步商品** — 只读查看（店铺管理不可删除/下架同步商品）
2. **改价** — 可修改售价，但不得低于 `min_price`
3. **自营商品** — 可自主上架（`sync_from_bid=NULL`），功能与普通商户一致

### 5.3 前端展示
- 直营店商品页展示最终售价
- 同步商品来源标记（可选）

---

## 六、非功能要求
- 直营店下单走直营店自己的订单流程（不改订单逻辑）
- 销量各自独立统计
- 库存各自独立管理
